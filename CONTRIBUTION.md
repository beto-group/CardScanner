# Contribution Guidelines

We welcome contributions to this Datacore component!

## Local Development Workflow

1. This component runs locally within the Datacore environment.
2. Ensure you have the `Datacore` plugin active in Obsidian.
3. Make changes in `src/index.jsx` or the files under `src/components/`, `src/styles/` or `src/utils/`.
4. Reload the note `CARD SCANNER.md` to re-run and hot-reload changes.

## Pull Request Process

1. Ensure the code is properly formatted and adheres to BetoOS styling (no emojis in UI, use Obsidian CSS variables, dynamic accent colors).
2. Do not commit any personal environment paths or secrets.
3. Update `README.md` if the component's features or directory structure change.
4. Submit your PR against the `main` branch.
